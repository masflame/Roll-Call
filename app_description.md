 — App Description

Overview
- Roll Call is a lecturer-focused attendance and session management platform designed for higher-education classrooms. Lecturers create time-bound sessions tied to modules, offerings (academic year/term/programme/year-level), and student groups. Students check in via QR/scan or short codes; attendance events and rich metadata are recorded in Firestore for analytics, audit, and compliance exports.

Core Features
- Session Management: Fast-create sessions linked to Module → Offering → Group, with options for live/recorded delivery and session privacy.
- Lightweight Integrity Fingerprinting: Each attendance submission includes a compact fingerprint (user agent, screen resolution, timezone) and IP metadata. The backend hashes these and stores an audit preview to enable proxy/duplicate detection without heavy device profiling.
- Proxy & Duplicate Detection: Server-side checks flag multiple submissions from the same fingerprint or IP and surfaces integrity warnings in real time to lecturers (banners and investigative modals).
- Lecturer UI & Workflows: A streamlined Create Session flow, live session dashboard with integrity indicators, session details/history pages, and module-level analytics for delivery and attendance trends.
- Compliance Exports: Enterprise-grade export bundles (CSV files + README) packaged as a ZIP for audits and record-keeping. Exports support filters (date ranges, modules, offerings, groups, student number) and include metadata (export id, generated by, scope, system version) to establish provenance.
- Admin CRUD: Admin pages for Modules, Offerings, and Groups so institutions can map curriculum structure and group cohorts precisely.

Architecture & Stack
- Frontend: React + TypeScript + Vite; Tailwind-like utility classes. Shared UI primitives (Card, Button, Input) ensure consistent UX and faster page updates.
- Backend: Firebase Firestore for data, Firebase Cloud Functions (TypeScript) for business logic: attendance submit endpoint, integrity checks, export generation (CSV → ZIP via JSZip), and analytics aggregation.
- Data Model Highlights: collections for `modules`, `offerings`, `groups`, `sessions`, `sessionsPrivate`, `sessionStats`, `moduleStats`, and per-session `attendance` and `integrity` subcollections.

Novelty & Value Proposition
- Lightweight, privacy-conscious fingerprinting: The app collects only a compact preview and a hash for detection and investigative contexts—balancing integrity needs with student privacy.
- Module → Offering → Group modeling: Many attendance tools treat modules as flat; Roll Call models academic structure (programme, academic year, term, year-level, cohort group) making scheduling, reporting, and compliance exports more precise and institution-friendly.
- Lecturer-first ergonomics: Fast session creation and clear, real-time integrity feedback reduce friction for instructors and make on-the-fly decisions feasible during live teaching.
- Compliance-focused exports: Built-in zipped export bundles with multiple normalized CSVs and README metadata simplify external audit, accreditation reporting, and institutional record-keeping.

Security, Privacy & Operational Notes
- Auditability: Every attendance event contains an `audit` object with hashed fingerprint and a small, human-readable preview so auditors can inspect suspicious events without exposing full device strings.
- Rate-limiting & abuse controls: Server-side rate limiting and proxy detection reduce automated or bulk-fraud attempts.
- Deployment: Designed for Firebase hosting + Cloud Functions; export bundles are generated server-side and returned as base64 ZIPs (or can be stored and handed via signed URLs for larger datasets).

Who should use it
- Universities and colleges needing a pragmatic, privacy-aware attendance solution with structured curriculum modelling, integrity detection, and exportability for compliance.
